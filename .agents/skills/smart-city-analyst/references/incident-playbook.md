# Incident Playbook Template

Используй для нового риск-типа (например, перед тем как добавить четвёртый сценарий). Один плейбук = один класс инцидентов, не один объект.

## Risk class

- Название: напр. «Разгерметизация теплового ввода»
- Домены ущерба: люди / инфраструктура / услуги / экология / публичная безопасность
- Типовая длительность: от первого сигнала до стабилизации
- Типовые жертвы (кого затрагивает в первую очередь)

## Telemetric footprint

Что физически меняется в объекте и окружении:

- Первичные признаки (что замечает ближайший датчик)
- Вторичные признаки (что проявляется через 5–15 минут)
- Ложные срабатывания (что похоже, но инцидентом не является)

## Source mix

Минимально достаточная комбинация источников для `elevated`:

- ≥1 sensor с триггером
- ≥1 external / virtual для подтверждения контекста
- actuator-обратная связь при локализации

## Criticality triggers (конкретные)

| Уровень | Триггер | Количественный порог (индикативно) | Длительность подтверждения |
|---|---|---|---|
| watch | | | ≥ 60 сек |
| elevated | | | ≥ 2 источника одновременно |
| high | | | факт угрозы людям / инфраструктуре |
| critical | | | массовый эффект / внешнее воздействие |

Числа — индикативные, помечаются как «оценка», финальная калибровка — за профильным ведомством.

## Dispatch roster

| Роль получателя | Когда подключается | Канал | SLA на приёмку | Условие приёмки |
|---|---|---|---|---|
| Аварийная бригада объекта | `elevated` | внутренняя связь | 2 мин | доклад на месте |
| Профильная городская служба | `elevated→high` | 112 / прямой канал | 5 мин | выезд подтверждён |
| Руководитель объекта | `elevated` | smartphone pult | 3 мин | подтверждение получено |
| Руководитель города | `high→critical` | leader view | 5 мин | решение принято |
| Публичные каналы | `critical` | пресс-штаб | 15 мин | заявление выпущено |

## Mitigation actuators

- Какие исполнительные устройства могут локализовать угрозу
- В каком порядке и кто имеет право их дёрнуть
- Какой feedback-источник подтверждает срабатывание

## Public safety layer

- Нужна ли эвакуация, радиус, маршруты
- Нужна ли информация для жителей, через какой канал
- Юридические/регуляторные последствия (уведомления надзорных органов)

## Post-incident

- Что фиксируется в журнале для разбора
- Какие метрики участвуют в KPI смены
- Какие уроки возвращаются в конфигурацию порогов и плейбука
