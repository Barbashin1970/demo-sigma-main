# Copy Voice for Sigma

Ограничения речи для всего русскоязычного UI Sigma: карточки, таймлайн, рекомендации, задачи, explainability, forecast.

## Регистр

- Деловой, сухой, предметный
- От третьего лица или безличный, без «мы» и без «вам»
- Глаголы в начале пункта: «закрыть», «оповестить», «проверить», «подтвердить»
- Без восклицаний, без эмодзи, без капслока

## Как аналитик пишет в журнал

Хорошо:

- «Сработал дымовой датчик в серверной 3-А, температура 68 °C, выросла за 4 минуты»
- «Закрыть задвижку 2-ТВ-07, оповестить старшую смены терапии, приостановить плановый забор воды»
- «Две независимых метки подтвердили превышение, гипотеза утечки холодоносителя отвергнута»

Плохо:

- «Внимание! Критическая ситуация!!!»
- «Наша умная система обнаружила проблему»
- «Пожалуйста, как можно скорее отреагируйте»
- «Революционный подход к управлению инцидентами»

## Запрещённая лексика

- «Бесшовно», «революционно», «следующее поколение», «экосистема»
- «Умный», «интеллектуальный» как рекламные эпитеты (если не в официальном названии «Умный город»)
- «Просто», «легко», «всего в один клик»
- Маркетинговые слоганы, призывы, вовлечение

## Единицы и числа

- Числа в моноширинном шрифте (`font-mono`), если в UI это метрика
- Единицы по-русски: °C, %, мин, м³/ч, мм рт. ст., ppm, м/с
- Десятичный разделитель — запятая (`47,2 %`), тысячный — неразрывный пробел (`12 400`)
- Время в формате `чч:мм` локального времени, дата `ДД.ММ.ГГГГ` когда нужна
- Не округлять до круглых чисел без нужды — `47,2 %` правдоподобнее `50 %`

## Роли и получатели

- Всегда по роли, не по имени: «старшая смены терапевтического отделения», «главный инженер»
- Ведомства — по общепринятому названию: «МЧС», «Теплосеть», «Росгидромет», «Водоканал»
- Внутренние объекты — по техническому идентификатору, если он есть: «задвижка 2-ТВ-07», «серверная 3-А»

## Формулировки по уровню критичности

| Уровень | Тон |
|---|---|
| watch | «зафиксировано отклонение», «запрошено подтверждение», «контроль по второй метке» |
| elevated | «подтверждено», «запущены превентивные задачи», «оповещены службы» |
| high | «реализована угроза X», «эскалация принята», «направлен наряд Y» |
| critical | «массовый риск», «введён режим ЧС», «публичное оповещение выпущено» |

## Explainability

Одна-две фразы. Структура: источник → факт → вывод → что отвергнуто.

Пример: «Датчик Т-14 показал +23 °C за 6 минут, подтверждено видеоаналитикой серверной 3-А. Версия сбоя датчика отвергнута: соседний Т-15 согласуется. Классификация — термический инцидент, уровень high.»

## Forecast

Две ветки, одинаковые единицы, одинаковый горизонт.

Пример: «Без вмешательства через 45 мин — возгорание соседнего стойкового ряда, простой сервисов до 8 ч. С автоматическим сбросом питания и пуском тушения — локализация в пределах одной стойки, простой до 40 мин.»

## Copy-review checklist

- [ ] Ни одного маркетингового штампа
- [ ] Каждая рекомендация — глагол + объект + ограничение
- [ ] Каждое число — с единицей, с запятой как разделителем
- [ ] Роли указаны по должности, не по имени
- [ ] Explainability содержит отвергнутую гипотезу
- [ ] Forecast с/без в одних единицах и с одним горизонтом
